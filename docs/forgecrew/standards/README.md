# Project Standards

This directory contains project-specific coding standards and conventions.

## Contents

| File             | Purpose                                        |
| ---------------- | ---------------------------------------------- |
| `API.md`         | API design patterns, endpoints, error handling |
| `CODING.md`      | General coding standards and patterns          |
| `DATABASE.md`    | Database conventions, queries, migrations      |
| `DEVELOPMENT.md` | Development environment and workflow           |
| `PERMISSIONS.md` | Claude Code permission configuration           |
| `SECURITY.md`    | Security requirements and patterns             |
| `TESTING.md`     | Testing guidelines and coverage requirements   |

## Structure

Each standards file should include:

1. **Purpose** - What this standard covers
2. **Rules** - The actual conventions to follow
3. **Examples** - Good and bad examples
4. **Exceptions** - When rules can be broken

---

_Customize these standards for your project's needs._
